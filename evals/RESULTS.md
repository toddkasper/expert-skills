# Eval Scoreboard

Per-skill competence results. One row per assessment run — **history is kept**, so lift and
scores are trendable per skill across content/curation cycles. Protocol:
[run-eval.md](run-eval.md). Method and thresholds: [../EVALS.md](../EVALS.md).

**Publish-ready:** skilled knowledge pass rate **≥ 85%** AND lift **> 0**. A skill whose lift
trends toward zero is restating what models already know natively — the signal to deepen (new
scars) or retire content, not to celebrate.

Columns: **K** = knowledge eval (`situations.md`, Lens 3). **App** = application eval
(`tasks.md`, Lens 4). **Rubric** = 12-dimension static audit total /36. **Trig** = trigger-test
pass rate (Lens 2). Leave cells `—` until that lens is run.
